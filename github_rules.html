üå≥ GitHub Branching Strategy

This model uses `main` for production, `dev` for integration, and task-specific branches for new work.

1. main Branch

Purpose: Always contains production-ready, stable code.

Protection: Highly protected. Merges only from `dev` after thorough testing.

Releases: Tagged for each release (e.g., v1.0.0).

2. dev Branch

Purpose: Contains the latest integrated development code.

Source: Created from `main`.

Integration: All specific task branches are merged into `dev`.

3. Specific Task Branches

Purpose: Dedicated branches for individual features, bugs, or updates, branching from `dev`.

Naming Convention:

  • feat/IssueNum-short-desc
  • bug/IssueNum-short-desc
  • up/IssueNum-short-desc
